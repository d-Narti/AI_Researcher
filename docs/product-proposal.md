## Обоснование идеи 

Современные научные исследования в областях STEM (математика, физика, химия, биология, информатика) всё чаще требуют написания программного кода для:

* численного моделирования
* анализа данных
* проверки гипотез
* воспроизводимости экспериментов

При этом значительная часть времени исследователя уходит не на саму научную гипотезу, а на:

* формализацию задачи
* декомпозицию решения
* написание вспомогательного кода

Предлагаемая система представляет собой агентную LLM-архитектуру, предназначенную для автоматизации этих этапов.

Система принимает научную задачу или исследовательский вопрос в естественном языке и автоматически формализует и декомпозирует задачу, после чего генерирует программную реализацию решения и предоставляет готовый код, пригодный для дальнейшего использования в исследовании.

Таким образом проект выступает инструментом ускорения исследовательского workflow, позволяя ученым быстрее переходить от постановки задачи к эксперименту.

## Цель

Целью является создание агентной систему, которая ускоряет научный процесс за счёт автоматизации этапов:

* формализации задачи
* алгоритмической декомпозиции
* генерации экспериментального кода.

## Метрики 

Продуктовые метрики:

* Research time reduction — сокращение времени от формулировки задачи до готового кода

Агентные метрики:

* качество декомпозиции задачи
* улучшение метрик генерации при использовании декомпозиции
* стабильность workflow (доля успешных pipeline)

Технические метрики:

* pass@1 на тестовых задачах
* latency на полный workflow
* доля runtime ошибок

## Потенциальные сценарии использования 

1) Алгоритмические и математические задачи

    Исследователь формулирует алгоритмическую проблему.

    Пример:

    Given an integer x, determine whether it is a palindrome.

    Система:

    * декомпозирует задачу
    * предлагает алгоритм
    * генерирует код

    Использование:

    * разработка алгоритмов
    * тестирование идей
    * обучение


2) Генетика и биология

    Исследователь формулирует генетическую задачу.

    Пример:

    Скрещивание двух гетерозиготных растений гороха (Aa × Aa).

    Система может:

    * формализовать модель наследования
    * построить решётку Пеннета
    * написать код для генерации распределений фенотипов.

3) Химия

    Пример:

    Определить относительную молекулярную массу H₂SO₄.

    Система может:

    * извлечь атомные массы
    * построить вычислительную модель
    * написать код для автоматического расчёта.

4) Физическое моделирование

    Пример:

    Три одинаковых заряда расположены в вершинах равностороннего треугольника.

    Система может:

    * формализовать задачу через закон Кулона
    * вывести систему уравнений
    * написать код для численного решения.

Edge cases:

* задачи без чёткой формализации
* противоречивые условия

## Ограничения 

**Latency**

Полный workflow:

> task → decomposition → code generation

Оценка:

| этап	| время |
| :--- | :--- |
| decomposition | 2–5 s |
| code generation | 5–10 s |

p95 latency ≈ 15–20 секунд

**Надёжность**

Основные проблемы:

* галлюцинации LLM
* некорректная формализация задачи
* синтаксические ошибки кода

**Операционные ограничения**

Стоимость зависит от:

* числа LLM вызовов
* размера промптов
* модели

Средняя стоимость на задачу:

~2 LLM calls
~1500–3000 tokens

## Архитектурный набросок 

Основные компоненты системы:

1. Agent Router

    Центральный оркестратор:
    * управляет последовательностью агентов
    * передаёт состояние между ними

2. Decomposer Agent

    Функции:

    * анализ научной задачи
    * разбиение на подзадачи
    * формирование плана решения

    Выход:
    * структурированный план

3. Code Generator Agent

    Функции:

    * генерация программного решения
    * использование декомпозиции

    Выход:

    * исполняемый код

## Потенциальный data flow 

Основной поток данных:

> Research Question → Decomposer Agent → Structured Plan → Code Generator Agent → Executable Code → Final Research Output

Делегируется LLM: 

* формализация задачи
* декомпозиция
* вывод алгоритма
* генерация кода

Не делегируется LLM:

* управление workflow
* контроль безопасности
* логирование
