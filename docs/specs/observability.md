# Observability / Evals спецификация

## 1. Роль

Обеспечивает:
- мониторинг
- логирование
- оценку качества системы

## 2. Логирование

Логируется:

- шаги workflow
- решения AgentRouter
- ошибки тестов
- success_rate

Формат:
- текстовые логи

## 3. Метрики

Основные:

- success_rate
- количество итераций
- время выполнения
- количество ошибок

## 4. Трейсинг

Минимальный:

- последовательность шагов
- история действий агента

## 5. Evals

Проверяется:

- корректность кода (через тесты)
- стабильность (повторяемость)
- эффективность (количество итераций)

## 6. Точки контроля

- после генерации кода
- после тестирования
- после оптимизации
- финальный результат

## 7. Ограничения

- нет внешних систем мониторинга
- нет алертов
- только локальные логи
