# Tools / API спецификация

## 1. Общая роль

Tools слой отвечает за:
- выполнение кода
- взаимодействие с LLM
- обеспечение безопасности выполнения

## 2. LLM (через LangChain)

### Контракт

Input:
- prompt (строка)

Output:
- text (строка)

### Использование

- DecomposerAgent
- CodeGeneratorAgent
- TestingAgent
- OptimizationAgent
- AgentRouter (принятие решений)

### Ограничения

- latency зависит от модели
- возможны нестабильные ответы
- ограничение контекста

## 3. Subprocess (выполнение кода)

### Контракт

Input:
- python файл
- stdin

Output:
- stdout
- stderr
- return code

### Ограничения

- timeout = x секунд
- выполнение изолировано через tempfile


## 4. AST анализ (безопасность)

### Проверки

Запрещены:
- subprocess
- socket
- requests
- os.system
- динамические импорты

### Поведение

- при обнаружении → выполнение блокируется
- ошибка записывается в test_results

## 5. Tempfile

Используется для:
- создания временных файлов с кодом
- изоляции исполнения


## 6. Guardrails

- ограничение времени выполнения
- проверка импортов
- ограничение числа тестов
- очистка временных файлов

## 7. Failure modes

- зависание процесса
- небезопасный код
- некорректный stdout
- ошибка выполнения

Fallback:
- ошибка фиксируется
- управление возвращается в AgentRouter
