# Retriever Spec

## 1. Статус в текущей версии

Retrieval-контур **отсутствует**.

## 2. Обоснование отсутствия

Задачи, решаемые системой, являются самодостаточными:
- Они описаны в тексте запроса с достаточным контекстом
- Не требуют поиска по внешним источникам (документации, кодовой базе, датасетам)

Добавление ретривера на этом этапе увеличило бы сложность без соразмерного прироста качества.

## 3. Когда ретривер стал бы необходим

| Сценарий | Нужен ретривер? |
|----------|----------------|
| Задачи по стандартным алгоритмам | Нет |
| Задачи по научным формулам (физика, химия) | Нет |
| Работа с конкретной кодовой базой пользователя | Да |
| Задачи по внутренней документации/API | Да |
| Работа с научными статьями или датасетами | Да |

## 4. Минимальная будущая реализация

При добавлении ретривера:

**Источники:**
- Документация Python-библиотек (NumPy, SciPy, pandas)
- Stack Overflow (топ-ответы)
- Примеры научного кода

**Индекс:**
- Vector store: FAISS или Chroma
- Embedding model: text-embedding-3-small или локальная модель

**Интеграция:**
- Подключение к `DecomposerAgent` (поиск по задаче перед декомпозицией)
- Подключение к `CodeGeneratorAgent` (поиск примеров кода)

**Reranking:**
- Cosine similarity по эмбеддингам
- При необходимости — cross-encoder reranker

**Ограничения:**
- Добавляет 0.5–2с latency на retrieval-шаг
- Требует поддержки индекса в актуальном состоянии
