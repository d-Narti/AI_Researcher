# AI Researcher

## Что за задача?

Основная задача — автоматизация рутинных этапов научного программирования с применением мультиагентной LLM-архитектуры. Система декомпозирует задачу, генерирует код, тестирует и оптимизирует его без участия пользователя.

## Для кого?

Для исследователей в областях STEM (математика, физика, химия, биология, информатика), которые регулярно пишут код для численного моделирования, анализа данных и проверки гипотез.

## Какие боли?

В текущем исследовательском процессе до **50% рабочего времени** уходит не на саму научную гипотезу, а на инфраструктуру вокруг неё:

### 1. Формализация задачи — 20–40% времени

- Исследователь имеет неформальную идею, которую нужно точно описать алгоритмически
- Каждая итерация: написать промпт → оценить результат → переформулировать — занимает 30–60 минут
- Ошибка формализации обнаруживается только на этапе тестирования, и весь цикл начинается заново
- Отсутствует стандартный шаблон перехода от гипотезы к алгоритму

### 2. Декомпозиция решения — 10–20% времени

- Сложную задачу необходимо разбить на изолированные подзадачи вручную
- Неправильная декомпозиция (например, пропущен шаг нормализации) приводит к полному переписыванию кода
- Каждый исследователь делает это по-своему — структура непредсказуема и не воспроизводима

### 3. Написание вспомогательного кода — 15–25% времени

- Boilerplate (парсинг аргументов, логирование, обработка ошибок) отнимает время, не добавляя ценности
- Тесты пишутся вручную и зачастую покрывают только happy path
- Численные эксперименты требуют большого количества повторяющегося шаблонного кода

## Что именно сделает PoC на демо?

Пользователь вводит научную задачу в свободной форме, например:

> "Simulate Lotka-Volterra predator-prey system with initial populations: prey=40, predators=9, alpha=0.1, beta=0.02, gamma=0.4, delta=0.02"

Система автоматически выполняет полный pipeline:

| Этап | Агент | Время | Результат |
|------|-------|-------|-----------|
| Формализация и план | DecomposerAgent | ~5 с | Пошаговый план решения |
| Генерация кода | CodeGeneratorAgent | ~15 с | Готовый Python-скрипт |
| Тестирование | TestingAgent | ~10 с | Результаты 3–5 тест-кейсов |
| Оптимизация (если нужно) | OptimizationAgent | ~10–20 с | Исправленный код |
| Финальный ответ | FormatterAgent | ~5 с | Структурированное решение |

**Итог за ~40–60 секунд:**
- Декомпозиция задачи (математическая модель, ODE-система, численное интегрирование)
- Исполняемый Python-код (~40–70 строк)
- Результаты тестирования (`3 passed / 3 total, success_rate: 1.0`)
- Объяснение решения на языке задачи

## Что НЕ делает PoC

- Проработанный пользовательский интерфейс (только базовый Streamlit)
- Визуализацию результатов и графиков
- Долгосрочную память между сессиями
- Гарантию 100% корректности решения
- Работу с задачами, требующими внешних данных или нескольких файлов

## Быстрый старт

В репозитории загружено [короткое демо](https://github.com/d-Narti/AI_Researcher/tree/main/ai4s_agent_system/short_demo.mkv), чтобы можно было посмотреть как выглядит поднятый интерфейс и что в нем отображается. Также там показаны поднятая модели, отрабатывающий celery с логами и api с запросами.
Если вы хотите запустить AI Researcher у себя локально, то вот следующие инструкции:

### Требования

| Инструмент | Версия | Зачем |
|-----------|--------|-------|
| Python | 3.10+ | Запуск кода агентов |
| [Docker Desktop](https://www.docker.com/products/docker-desktop/) | любая | RabbitMQ + Redis |
| [Ollama](https://ollama.com/) | любая | Локальная LLM |

### 1. Клонировать репозиторий

```bash
git clone <repo-url>
cd AI_Researcher/ai4s_agent_system
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Создать файл конфигурации

```bash
cp env.example .env
```

Содержимое `.env` по умолчанию уже настроено на локальный Ollama — менять ничего не нужно:

```env
OPENAI_API_BASE=http://localhost:11434/v1
OPENAI_API_KEY=ollama
CELERY_BROKER_URL=amqp://admin:admin@localhost:5672/
CELERY_RESULT_BACKEND=redis://localhost:6379/0
```

### 4. Скачать модель в Ollama


Можно установить любую модель, для примера я установил маленькую gemma4 квантованную на 8b параметров. Данная модель хорошо справляется с декомпозицией и написанием кода, но могут возникать проблемы при тестировании кода, зачастую модель не справляется именно с форматом входных и выходных данных, в связи с чем парсер откидывает верные, но записанные в формате, невозможном для стабильного отслеживания и парсинга, ответы. Это как раз показано в демо видео, видно что модель сформировала неверные входные данные для одного теста и ожидала ответа. Модель на 26b параметров справляется намного лучше и успешно справляется со всеми этапами, но, к сожалению, не везде можно локально раскатить такую большую модель.

```bash
ollama pull gemma4:e4b
```

> Убедитесь, что Ollama запущена (`ollama serve`). При установке Ollama Desktop она стартует автоматически.

### 5. Запустить инфраструктуру (RabbitMQ + Redis)

```bash
docker compose up rabbitmq redis -d
```

Дождаться статуса `healthy` (10–20 секунд). Проверить: `docker compose ps`.

### 6. Запустить сервисы

Откройте **3 отдельных терминала** в папке `ai4s_agent_system/`:

**Терминал A — Celery worker**
```bash
# Windows
celery -A src.celery.app worker --loglevel=info --pool=solo

# Linux / macOS
celery -A src.celery.app worker --loglevel=info
```

**Терминал B — FastAPI**
```bash
python api.py
# или: python run.py api
```

**Терминал C — Streamlit UI**
```bash
streamlit run streamlit_app.py
# или: python run.py streamlit
```

Открыть в браузере: [http://localhost:8501](http://localhost:8501)

### Проверка работоспособности

```bash
# API health check
curl http://localhost:8000/health
# → {"status": "healthy"}

# RabbitMQ management UI
# http://localhost:15672  (admin / admin)
```

### Альтернатива: полный Docker (все сервисы в контейнерах)

> Требует Ollama на хосте. В docker-compose.yml заменить `localhost` на `host.docker.internal` в переменной `OPENAI_API_BASE`.

```bash
docker compose up --build
```

Streamlit будет доступен на [http://localhost:8501](http://localhost:8501), API на [http://localhost:8000](http://localhost:8000).
